Casos de Uso

CU001 Listar Anuncis

Descripción: Permitir a los usuarios ver una lista de todos los anuncios disponibles.

Actores: Usuario

Precondiciones: El usuario debe estar en la página principal o en una página de anuncios.

Flujo Principal:

  1. El usuario accede a la sección de anuncios.
  2. El sistema muestra una lista de todos los anuncios disponibles.
  3. El usuario puede navegar a través de la lista y ver los títulos y descripciones breves de los anuncios.

Postcondiciones: El usuario visualiza los anuncios disponibles.

Excepciones: Si no hay anuncios disponibles, el sistema muestra un mensaje indicando que no hay anuncios para mostrar.

CU002 Veure Anunci

Descripción: Permitir a los usuarios ver los detalles completos de un anuncio específico.

Actores: Usuario

Precondiciones: El usuario debe estar visualizando la lista de anuncios.

Flujo Principal:

  1. El usuario selecciona un anuncio de la lista.
  2. El sistema muestra los detalles completos del anuncio seleccionado, incluyendo título, descripción, precio, imágenes, y datos de contacto del vendedor.

Postcondiciones: El usuario visualiza la información detallada del anuncio seleccionado.

Excepciones: Si el anuncio seleccionado ya no está disponible, el sistema muestra un mensaje de error y regresa a la lista de anuncios.

CU003 Buscar Anunci

Descripción: Permitir a los usuarios buscar anuncios específicos utilizando criterios de búsqueda.

Actores: Usuario

Precondiciones: El usuario debe estar en la sección de búsqueda de anuncios.

Flujo Principal:

  1. El usuario ingresa criterios de búsqueda (palabras clave, categoría, rango de precio, etc.).
  2. El usuario inicia la búsqueda.
  3. El sistema filtra y muestra los anuncios que coinciden con los criterios de búsqueda ingresados.

Postcondiciones: El usuario visualiza una lista de anuncios que coinciden con los criterios de búsqueda.

Excepciones: Si no se encuentran anuncios que coincidan con los criterios, el sistema muestra un mensaje indicando que no se encontraron resultados.

CU004 Contacte

Descripción: Permitir a los usuarios contactar al vendedor de un anuncio.

Actores: Usuario, Vendedor

Precondiciones: El usuario debe estar visualizando los detalles de un anuncio.

Flujo Principal:

  1. El usuario selecciona la opción de contactar al vendedor.
  2. El sistema muestra un formulario de contacto.
  3. El usuario llena el formulario con su mensaje y detalles de contacto.
  4. El usuario envía el mensaje.
  5. El sistema envía el mensaje al vendedor y muestra una confirmación al usuario.

Postcondiciones: El mensaje es enviado al vendedor.

Excepciones: Si hay un error en el envío del mensaje, el sistema muestra un mensaje de error y sugiere intentar nuevamente.

CU005 Registre

Descripción: Permitir a nuevos usuarios registrarse en el sistema.

Actores: Nuevo Usuario

Precondiciones: El usuario debe acceder a la página de registro.

Flujo Principal:

  1. El usuario selecciona la opción de registrarse.
  2. El sistema muestra un formulario de registro.
  3. El usuario completa el formulario con sus datos (nombre, correo electrónico, contraseña, etc.).
  4. El usuario envía el formulario.
  5. El sistema valida los datos y crea una nueva cuenta de usuario.
  6. El sistema muestra una confirmación de registro exitoso.

Postcondiciones: Se crea una nueva cuenta de usuario.

Excepciones: Si hay errores en los datos ingresados, el sistema muestra mensajes de error específicos y solicita correcciones.

CU006 Publicar Anunci

Descripción: Permitir a los usuarios registrados publicar un nuevo anuncio.

Actores: Usuario Registrado

Precondiciones: El usuario debe estar registrado y haber iniciado sesión.

Flujo Principal:

  1. El usuario accede a la sección para publicar un anuncio.
  2. El sistema muestra un formulario de publicación de anuncio.
  3. El usuario completa el formulario con los detalles del anuncio (título, descripción, precio, imágenes, etc.).
  4. El usuario envía el formulario.
  5. El sistema valida la información y publica el anuncio.
  6. El sistema muestra una confirmación de publicación exitosa.

Postcondiciones: El anuncio es publicado y visible para otros usuarios.

Excepciones: Si hay errores en los datos ingresados, el sistema muestra mensajes de error específicos y solicita correcciones.

CU007 Modificar Anunci

Descripción: Permitir a los usuarios registrados modificar un anuncio existente.

Actores: Usuario Registrado

Precondiciones: El usuario debe estar registrado, haber iniciado sesión, y ser el propietario del anuncio.

Flujo Principal:

  1. El usuario accede a la sección de sus anuncios.
  2. El usuario selecciona el anuncio que desea modificar.
  3. El sistema muestra el formulario de edición con los detalles actuales del anuncio.
  4. El usuario modifica los detalles del anuncio.
  5. El usuario envía el formulario modificado.
  6. El sistema valida la información y actualiza el anuncio.
  7. El sistema muestra una confirmación de modificación exitosa.

Postcondiciones: El anuncio es actualizado con la nueva información.

Excepciones: Si hay errores en los datos ingresados, el sistema muestra mensajes de error específicos y solicita correcciones. Si el usuario no es el propietario del anuncio, el sistema muestra un mensaje de error y no permite la modificación.

CU008 Eliminar Anunci

Descripción: Permitir a los usuarios registrados eliminar un anuncio existente.

Actores: Usuario Registrado

Precondiciones: El usuario debe estar registrado, haber iniciado sesión, y ser el propietario del anuncio.

Flujo Principal:

  1. El usuario accede a la sección de sus anuncios.
  2. El usuario selecciona el anuncio que desea eliminar.
  3. El sistema muestra una confirmación de eliminación.
  4. El usuario confirma la eliminación del anuncio.
  5. El sistema elimina el anuncio de la base de datos.
  6. El sistema muestra una confirmación de eliminación exitosa.

Postcondiciones: El anuncio es eliminado y ya no es visible para otros usuarios.

Excepciones: Si el usuario no es el propietario del anuncio, el sistema muestra un mensaje de error y no permite la eliminación. Si hay un error en el proceso de eliminación, el sistema muestra un mensaje de error y sugiere intentar nuevamente.

CU009 Home

Descripción: Permitir a los usuarios acceder a la página principal del sitio.

Actores: Usuario

Precondiciones: El usuario debe acceder al sitio web.

Flujo Principal:

  1. El usuario accede a la página principal del sitio web.
  2. El sistema muestra el contenido de la página principal, que incluye:
    • Una barra de navegación con enlaces a diferentes secciones del sitio.
    • Un buscador para buscar anuncios.
    • Listado destacado de anuncios recientes o populares.
    • Enlaces para registrarse o iniciar sesión (si el usuario no está autenticado).
    • Enlaces a las secciones de publicar anuncio, ver perfil, etc. (si el usuario está autenticado).

Postcondiciones: El usuario visualiza la página principal del sitio con las opciones de navegación disponibles.

Excepciones: Si hay problemas técnicos que impiden cargar la página principal, el sistema muestra un mensaje de error y posibles soluciones (como recargar la página o intentar más tarde).
